# Astel — Design Context

> Codename AURIGA → product name **Astel**. Layered Gaussian-splat generation platform
> (Meshy-AI competitor). Package format `.astel`. This file is the design source of truth
> for the web app (`apps/web`).

## Design Context

### Users
3D artists, technical artists, game/VFX studios, and capture professionals who currently
use Meshy/Tripo/Luma but distrust "plausible mush" output. They work in dim studio
environments, often alongside Blender/Unreal/Houdini. They judge a tool in the first 30
seconds by how its viewport feels. The job: turn a prompt, photo, or video into a
*measurably accurate*, world-aware splat asset they can drop into an engine — and to be
able to *see* which parts are measured vs. generated (the Truth Meter).

### Brand Personality
Three words: **instrument · honest · cinematic.** Astel is a precision instrument, not a
toy generator. It never hides uncertainty (the confidence channel is "sacred"). The
interface should feel like the control surface of a high-end capture rig or a film color
suite — calm, dark, dense with real signal, zero hype, zero generic-AI gloss.

### Aesthetic Direction
- **Theme: dark.** Derived from context — used in dim studios next to 3D DCC tools at
  night; the viewport's splats are the light source, the UI recedes around them.
- **Anti-references:** the cyan-on-black / purple-blue-gradient "AI SaaS" look; neon glow
  borders; glassmorphism everywhere; rounded-icon-above-every-heading templates. Astel
  must NOT look like a 2024 AI startup.
- **References in spirit:** Linear's restraint, a Teenage Engineering device's labeled
  precision, a film-grade scope/waveform panel, observatory instrument readouts.
- **Signature element ("the one thing you remember"):** the **Layer Stack rail** — L0→L7
  rendered as an instrument column on the right, each layer a precise readout with its own
  availability state, that you can scrub like a film timeline. Plus the **Truth Meter**:
  honest measured-vs-generated numbers no competitor dares show.

### Palette (OKLCH, dark)
- Brand hue ≈ **62° (amber/brass)** — an instrument/brass accent, deliberately NOT blue or
  purple. Used at ~10% weight: active layer, focus rings, key CTA, live progress.
- Neutrals tinted toward the brand hue (chroma ~0.006–0.012) — warm near-black surfaces,
  never pure #000/#fff. Surface elevation by lightness (not shadow): bg 14% → panel 17% →
  raised 21%.
- Semantic: measured/confident = a cool teal (≈195°) used ONLY in the Truth Meter and
  confidence channel, so "measured reality" reads as a distinct, trustworthy signal vs the
  brass "generated/active" accent. Error = restrained red (≈25°).

### Typography
- **Display / UI labels:** `Space Grotesk` is banned by the skill — use **"Archivo"**
  (variable grotesk, technical but warm) for headings and instrument labels.
- **Body / data:** **"Spline Sans"** for UI body; **"Spline Sans Mono"** for all numeric
  readouts, metrics, splat counts, coordinates (tabular-nums). Mono is used *as data*, not
  as lazy "developer vibes" decoration.
- Fixed `rem` scale (this is a product UI, not a marketing page). 1.25 ratio.

### Design Principles
1. **The viewport is the hero; the UI is instrumentation around it.** Chrome stays dark,
   quiet, and out of the way. Splats are the only thing that glows.
2. **Honesty is visible.** Confidence, measured-vs-generated, and locked/pending layer
   states are first-class UI, never hidden. The Truth Meter is a flagship surface.
3. **Everything reads like an instrument.** Precise labels, tabular numeric readouts,
   real units. No decorative sparklines, no fake dashboards.
4. **Motion is functional only.** Staged pipeline progress, layer transitions, focus —
   transform/opacity, exponential ease-out, full reduced-motion support. Nothing bounces.
5. **No AI-slop tells.** No gradient text, no side-stripe accent borders, no glow-glass.
   If it looks like an AI made it, it's wrong.
